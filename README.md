# 🌐 Multi-Threaded File Server (C++ | Linux | POSIX Threads)

A high-performance **multi-threaded file server** built in **C++** that demonstrates **Operating System concepts**, **Inter-Process Communication (IPC)**, and **Concurrent Programming** using **POSIX Threads, Shared Memory, and Semaphores**.

The server efficiently handles multiple client requests in parallel, ensuring synchronized access to shared resources while maintaining data consistency and performance.

# 📌 Project Overview

The Multi-Threaded File Server allows multiple clients to simultaneously request specific lines from text files stored on the server.

Each client generates multiple requests, while the server processes them concurrently by spawning worker threads.

Communication between the server and clients is implemented using:

- **POSIX Shared Memory** for fast data exchange
- **POSIX Semaphores** for synchronization
- **POSIX Threads (pthreads)** for concurrent request handling

This project demonstrates practical implementation of:

- Operating Systems
- Concurrent Programming
- Thread Synchronization
- Linux IPC
- File System Operations

---

# ⚙️ How It Works

1. The server starts and creates worker threads.
2. Clients send requests for specific file lines.
3. Each request is assigned to a thread.
4. The server reads the requested data.
5. Results are written into shared memory.
6. Semaphores synchronize read/write operations.
7. The client receives the requested data.
8. Performance metrics are recorded.

# ▶️ Running the Server

```bash
./fileServer <num_clients> <num_requests> <lambda>
```

Example

```bash
./fileServer 5 100 10
```

Where

- `num_clients` → Number of concurrent clients
- `num_requests` → Requests generated by each client
- `lambda` → Delay parameter used during request generation
